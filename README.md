### Travel Wallet Telegram Bot

## 📋 О проекте

**Цель:**  
Помочь путешественнику вести баланс в двух валютах (родная и местная) и отслеживать траты в реальном времени без ручного пересчёта.

**Результат:**  
Рабочий Telegram-бот с поддержкой нескольких поездок, двойным балансом и историей операций. Данные сохраняются локально.

**Ключевые задачи:**
- Интеграция с внешним API курсов валют (exchangerate.host).
- Проектирование структуры базы данных SQLite для хранения транзакций.
- Разработка удобного inline-меню для навигации без ввода команд.
- Обработка ошибок сети и некорректного ввода данных.

---

### Основные возможности

- **Создание путешествия**:
  - Указываете страну отправления и страну назначения.
  - Бот определяет валюты стран (для популярных направлений) и проверяет курс через `http://api.exchangerate.host/convert`.
  - Показывает текущий курс и предлагает принять его или ввести свой (например, курс обменника).
  - Запрашивает начальную сумму в домашней валюте и сразу считает стартовый баланс в обеих валютах.
- **Несколько путешествий**:
  - Можно создать несколько путешествий и переключаться между ними.
  - Для каждого хранится валютная пара, курс, баланс и история расходов.
- **Учёт расходов**:
  - Любое текстовое сообщение, которое является числом, интерпретируется как расход в валюте страны пребывания.
  - Бот пересчитывает сумму в домашнюю валюту по сохранённому курсу и предлагает учесть расход.
  - При подтверждении расход записывается в SQLite и вычитается из баланса.
- **Всегда двойной баланс**:
  - Баланс показывается в двух валютах, например: `Остаток: 2400 CNY = 30720 RUB`.
- **Управление через inline‑меню и команды**:
  - Главное меню (inline‑кнопки):
    - `🧳 Создать путешествие`
    - `🌍 Мои путешествия`
    - `💰 Баланс`
    - `📊 История расходов`
    - `⚙️ Изменить курс`
  - Альтернативные слэш‑команды:
    - `/start` — приветствие и главное меню.
    - `/newtrip` — создать новое путешествие.
    - `/switch` — переключить активное путешествие.
    - `/balance` — показать текущий баланс.
    - `/history` — показать историю расходов.
    - `/setrate` — изменить курс для активного путешествия.

### Технические детали

- **Язык и библиотеки**:
  - Python 3.10+ (рекомендуется).
  - `pytelegrambotapi` — Telegram‑бот.
  - `requests` — HTTP‑запросы к API.
  - `python-dotenv` — загрузка конфигурации из `.env`.
- **API валют**:
  - Используется строго `api.exchangerate.host` и только endpoint:
    - `http://api.exchangerate.host/convert`
  - Все обращения к API инкапсулированы в `current_api.py`:
    - `convert_currency(amount, from_currency, to_currency)` — базовый вызов `/convert`.
    - `get_rate(from_currency, to_currency)` — получение курса (rate) через `/convert` при `amount=1`.
- **Хранение данных**:
  - Локальная база SQLite `travel_wallet.db`.
  - Таблицы:
    - `users` — Telegram‑пользователи и активное путешествие.
    - `trips` — путешествия (страны, валюты, курс, баланс).
    - `expenses` — расходы по каждому путешествию.
  - Каждый Telegram‑пользователь имеет собственный набор путешествий и расходов.

### Установка и запуск

1. **Клонируйте/скачайте проект** и перейдите в папку:

```bash
cd "d:\Cursor proj\api currency 2"
```

2. **Создайте и активируйте виртуальное окружение (рекомендуется)**:

```bash
python -m venv venv
venv\Scripts\activate
```

3. **Установите зависимости**:

```bash
pip install -r requirements.txt
```

4. **Создайте файл `.env` в корне проекта (если ещё не создан)** и укажите:

```env
CURRENCY_API_KEY=ВАШ_БОЕВОЙ_API_КЛЮЧ_ОТ_api.exchangerate.host
TELEGRAM_TOKEN=ВАШ_ТОКЕН_ОТ_BotFather
```

- `CURRENCY_API_KEY` — боевой ключ, полученный в личном кабинете APILayer / Exchangerate.host  
  (см. документацию: [Exchangerate.host API](https://docs.apilayer.com/Exchangerate/docs/api-documentation?utm_source=ExchangerateHomePage&utm_medium=Referral)).
- `TELEGRAM_TOKEN` — токен вашего Telegram‑бота от BotFather.

5. **Запустите бота**:

```bash
python travel_wallet_bot.py
```

После запуска бот начнёт опрашивать Telegram (`infinity_polling`).  
Откройте ваш бот в Telegram и отправьте `/start`.

### Как это работает внутри

- Файл `current_api.py`:
  - Загружает `CURRENCY_API_KEY` из `.env`.
  - Все запросы делает на `http://api.exchangerate.host/convert` с параметром `access_key`.
  - Функция `get_rate` получает актуальный курс (поле `info.rate`) при `amount=1`.
- Файл `travel_wallet_bot.py`:
  - Загружает `TELEGRAM_TOKEN` и инициализирует `TeleBot`.
  - Настраивает SQLite‑базу (`init_db`).
  - Реализует FSM (простое хранение состояний в памяти) для диалога создания путешествия:
    - Ввод страны отправления → определение валюты → при необходимости запрос кода валюты.
    - Ввод страны назначения → определение валюты → при необходимости запрос кода.
    - Запрос курса через API и предложение принять/изменить.
    - Ввод начальной суммы в домашней валюте и создание записи в БД.
  - Обрабатывает:
    - Inline‑кнопки главного меню.
    - Кнопки подтверждения/отмены расходов.
    - Слэш‑команды как альтернативу меню.
  - Любое число вне режимов настройки интерпретируется как расход в валюте назначения.

### Обработка ошибок

- **Ошибки API**:
  - При проблемах с сетью или неуспешном ответе `get_rate` возвращает `None`, и бот просит ввести курс вручную.
- **Некорректный ввод пользователя**:
  - При вводе нечисловых значений там, где ожидается число (курс, сумма) — бот просит повторить ввод дружелюбным сообщением.
  - Если отправлено текстовое сообщение, которое не является числом, и бот не ждёт специальный ввод — он подсказывает использовать клавиши меню.
- **Отсутствие активного путешествия**:
  - При попытке показать баланс/историю или учесть расход без активного путешествия бот предлагает сначала создать его.
